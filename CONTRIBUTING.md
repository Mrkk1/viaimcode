## English

Thank you for your interest in contributing to viaimCode AI! We welcome contributions from everyone and are grateful for every contribution, no matter how small.

### 🚀 Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/your-username/viaimcode.git
   cd viaimcode
   ```
3. **Install dependencies**:
   ```bash
   npm install
   ```
4. **Set up your development environment** following the [README](README.md)

### 📋 Types of Contributions

We welcome several types of contributions:

- 🐛 **Bug Reports**: Help us identify and fix issues
- 💡 **Feature Requests**: Suggest new features or improvements
- 📝 **Documentation**: Improve our docs, tutorials, or code comments
- 🔧 **Code Contributions**: Fix bugs or implement new features
- 🌐 **Translations**: Help translate the project to other languages
- 🧪 **Testing**: Write tests or help with quality assurance

### 🐛 Reporting Bugs

Before submitting a bug report:

1. **Check existing issues** to avoid duplicates
2. **Use the latest version** to ensure the bug still exists
3. **Provide detailed information**:
   - Operating system and version
   - Node.js version
   - Browser (if applicable)
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots or error messages

### 💡 Suggesting Features

When suggesting new features:

1. **Check existing feature requests** first
2. **Explain the use case** and why it would be valuable
3. **Provide mockups or examples** if possible
4. **Consider the scope** - start with smaller, focused features

### 🔧 Code Contributions

#### Development Workflow

1. **Create a feature branch**:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes** following our coding standards

3. **Test your changes**:
   ```bash
   npm run lint
   npm run type-check
   npm run build
   ```

4. **Commit your changes** with a clear message:
   ```bash
   git commit -m "feat: add new feature description"
   ```

5. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Create a Pull Request** with a clear description

#### Coding Standards

- **TypeScript**: Use TypeScript for all new code
- **ESLint**: Follow the existing ESLint configuration
- **Prettier**: Code formatting is handled automatically
- **Components**: Use functional components with hooks
- **API Routes**: Follow Next.js App Router conventions
- **Commit Messages**: Use conventional commit format

#### Pull Request Guidelines

- **Clear title and description**
- **Reference related issues** (e.g., "Fixes #123")
- **Include screenshots** for UI changes
- **Update documentation** if needed
- **Keep changes focused** - one feature per PR
- **Ensure tests pass** before submitting

### 🧪 Testing

We use various testing approaches:

- **Unit Tests**: Test individual components and functions
- **Integration Tests**: Test API endpoints and workflows
- **E2E Tests**: Test complete user workflows
- **Manual Testing**: Test in different browsers and devices

### 📝 Documentation

Help improve our documentation:

- **README updates**: Keep installation and usage instructions current
- **Code comments**: Add helpful comments for complex logic
- **API documentation**: Document new endpoints and parameters
- **Tutorials**: Create guides for common use cases

### 🌐 Internationalization

Help make viaimCode AI accessible to more users:

- **Translate UI text**: Help translate the interface
- **Localize documentation**: Translate README and guides
- **Cultural adaptation**: Ensure features work well in different regions

### 🏆 Recognition

Contributors are recognized in several ways:

- **Contributors section** in README
- **Release notes** mention significant contributions
- **GitHub profile** gets contribution activity
- **Community recognition** in discussions and social media

### 📞 Getting Help

If you need help with contributing:

- **GitHub Discussions**: Ask questions and get help from the community
- **Issues**: Use the "question" label for general questions
- **Discord/Slack**: Join our community chat (if available)
- **Email**: Contact the maintainers directly for sensitive issues

### 📄 Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.

## 中文

感谢您对 viaimCode AI 项目的贡献兴趣！我们欢迎所有人的贡献，并对每一个贡献都心存感激，无论大小。

### 🚀 开始贡献

1. **Fork 仓库** 到您的 GitHub 账号
2. **克隆您的 fork** 到本地:
   ```bash
   git clone https://github.com/your-username/viaimcode.git
   cd viaimcode
   ```
3. **安装依赖**:
   ```bash
   npm install
   ```
4. **设置开发环境** 按照 [README](README-zh.md) 的说明

### 📋 贡献类型

我们欢迎多种类型的贡献：

- 🐛 **Bug 报告**: 帮助我们识别和修复问题
- 💡 **功能请求**: 建议新功能或改进
- 📝 **文档**: 改进文档、教程或代码注释
- 🔧 **代码贡献**: 修复 bug 或实现新功能
- 🌐 **翻译**: 帮助将项目翻译成其他语言
- 🧪 **测试**: 编写测试或帮助质量保证

### 🐛 报告 Bug

提交 bug 报告前：

1. **检查现有问题** 避免重复
2. **使用最新版本** 确保 bug 仍然存在
3. **提供详细信息**:
   - 操作系统和版本
   - Node.js 版本
   - 浏览器（如适用）
   - 重现步骤
   - 预期 vs 实际行为
   - 截图或错误信息

### 💡 建议功能

建议新功能时：

1. **先检查现有功能请求**
2. **解释用例** 和为什么它有价值
3. **提供模型或示例**（如可能）
4. **考虑范围** - 从小的、专注的功能开始

### 🔧 代码贡献

#### 开发工作流

1. **创建功能分支**:
   ```bash
   git checkout -b feature/你的功能名
   ```

2. **进行更改** 遵循我们的编码标准

3. **测试您的更改**:
   ```bash
   npm run lint
   npm run type-check
   npm run build
   ```

4. **提交更改** 使用清晰的消息:
   ```bash
   git commit -m "feat: 添加新功能描述"
   ```

5. **推送到您的 fork**:
   ```bash
   git push origin feature/你的功能名
   ```

6. **创建 Pull Request** 并提供清晰的描述

#### 编码标准

- **TypeScript**: 所有新代码使用 TypeScript
- **ESLint**: 遵循现有的 ESLint 配置
- **Prettier**: 代码格式化自动处理
- **组件**: 使用带 hooks 的函数式组件
- **API 路由**: 遵循 Next.js App Router 约定
- **提交消息**: 使用约定式提交格式

#### Pull Request 指南

- **清晰的标题和描述**
- **引用相关问题**（例如："Fixes #123"）
- **包含截图**（UI 更改）
- **更新文档**（如需要）
- **保持更改专注** - 每个 PR 一个功能
- **确保测试通过** 再提交

### 🧪 测试

我们使用多种测试方法：

- **单元测试**: 测试单个组件和函数
- **集成测试**: 测试 API 端点和工作流
- **端到端测试**: 测试完整的用户工作流
- **手动测试**: 在不同浏览器和设备中测试

### 📝 文档

帮助改进我们的文档：

- **README 更新**: 保持安装和使用说明最新
- **代码注释**: 为复杂逻辑添加有用的注释
- **API 文档**: 记录新的端点和参数
- **教程**: 为常见用例创建指南

### 🌐 国际化

帮助让 viaimCode AI 为更多用户可用：

- **翻译 UI 文本**: 帮助翻译界面
- **本地化文档**: 翻译 README 和指南
- **文化适应**: 确保功能在不同地区运行良好

### 🏆 认可

贡献者通过多种方式获得认可：

- **README 中的贡献者部分**
- **发布说明** 提及重要贡献
- **GitHub 个人资料** 获得贡献活动
- **社区认可** 在讨论和社交媒体中

### 📞 获取帮助

如果您需要贡献方面的帮助：

- **GitHub Discussions**: 向社区提问和获取帮助
- **Issues**: 使用 "question" 标签提出一般问题
- **Discord/Slack**: 加入我们的社区聊天（如可用）
- **邮箱**: 直接联系维护者处理敏感问题

### 📄 行为准则

请注意，此项目发布时附带贡献者行为准则。参与此项目即表示您同意遵守其条款。
